# 人生 K 线项目 - 面试项目简介

## 一、项目概述（30秒）

**项目名称**：人生 K 线（Life Destiny K-Line）  
**项目定位**：将传统八字命理学与现代金融可视化技术相结合的创新工具  
**项目地址**：https://lesliewylie.github.io  
**GitHub**：https://github.com/LeslieWylie/LeslieWylie.github.io

这是一个**全栈前端项目**，通过 AI 大模型分析用户的生辰八字，生成 100 年的人生运势 K 线图，帮助用户以直观的方式了解人生起伏、把握关键转折点。

---

## 二、项目背景与痛点（1分钟）

### 传统命理工具的三大痛点：
1. **输入繁琐**：排盘步骤复杂，不懂干支的人难以入手
2. **结果抽象**：输出以长文为主，缺少趋势视角，难以比较不同阶段
3. **结果难保存/复盘**：缺少结构化存储和历史对比功能

### 解决方案：
- 将复杂的排盘流程简化为 **3 步主路径**（填八字 → 生成 Prompt → 上传 JSON）
- 用 **K 线图**可视化 100 年运势，绿色代表吉运，红色代表凶运
- 设计 **6 大维度评分体系**（总评、事业、财富、婚姻、健康、六亲）
- 支持 **JSON/PNG/PDF 多格式导出**，方便保存和分享

---

## 三、核心功能与技术亮点（2-3分钟）

### 1. 智能 Prompt 生成系统
- **技术实现**：设计了"系统角色设定 + 用户提示词"双段式 Prompt 模板
- **多平台适配**：支持 Gemini、Kimi、豆包、千问等多个 AI 平台
- **用户体验**：一键复制系统提示、用户提示、完整对话三种模式，降低操作门槛
- **数据质量**：通过优化 Prompt 结构，将 AI 输出格式稳定率从 ~60% 提升到 ~95%

### 2. 多版本数据格式兼容
- **技术挑战**：不同 AI 模型返回的 JSON 格式不一致（V1/V2/V3/V4）
- **解决方案**：
  - 使用 **TypeScript + Zod** 进行类型定义和数据验证
  - 设计统一的数据转换层，自动识别并转换不同格式
  - 支持部分数据输出（如 V4 格式可能只返回前 20 年数据）
- **技术价值**：提高了系统的健壮性和可扩展性

### 3. 增强型 K 线图组件
- **技术栈**：React + Recharts + TypeScript
- **核心功能**：
  - 100 年运势可视化，支持缩放、筛选、关键年份标注
  - 鼠标滚轮缩放、双指捏合缩放（移动端）
  - 点击 K 线查看详细流年批断
  - 大运分界线自动标注
- **性能优化**：使用 `useMemo` 优化数据计算，支持大数据量流畅渲染

### 4. 数据闭环与增长机制
- **行为数据采集**：
  - 前端统一埋点服务（`usageLogger.ts`）
  - Vercel Serverless Function + Supabase 存储
  - 记录核心操作：生成 Prompt、上传结果、查看历史、反馈等
- **反馈机制**：
  - 结果页集成反馈入口
  - 结合埋点数据与用户反馈，迭代优化 Prompt 结构和交互流程
- **数据驱动迭代**：通过数据分析发现问题 → 调整 Prompt/交互 → 验证效果

## 四、技术栈与架构设计（1分钟）

### 前端技术栈
- **框架**：React 19 + TypeScript + Vite
- **UI**：Tailwind CSS + Lucide React（图标）
- **图表**：Recharts
- **表单**：React Hook Form + Zod 验证
- **导出**：html2canvas + jsPDF

### 后端/基础设施
- **部署**：GitHub Pages（前端）+ Vercel（Serverless Functions）
- **数据库**：Supabase（PostgreSQL）
- **日志服务**：自定义 Serverless API + Supabase 存储

### 架构特点
- **组件化设计**：15+ 个功能组件，职责清晰
- **类型安全**：完整的 TypeScript 类型定义
- **数据验证**：使用 Zod 进行运行时数据校验
- **错误处理**：ErrorBoundary + Toast 通知系统
